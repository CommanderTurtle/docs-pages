Welcome to sHEL¶

All the protection of a turtle without the soft underbelly.

Star Watch Fork

sHEL is a volatile, literal-safe, automation-friendly data substrate written for low-level communication.

Originally developed for using boolean structure in multi-line assembly, sHEL has evolved into a reliable communication infrastructure for automation and LLMs.

Where to Get Started¶

  • Run open-source automation without an LLM? Start with Flows
  • Build applications with sHEL using strongly-typed markdown, XML, and F#? See creating a Markdown Registry
  • Begin web development with F#? Learn the Sharp Orchestrator

What sHEL Is¶

sHEL can be used as:

  • A full schema system
  • A query language
  • A serializer/deserializer
  • A macro-friendly API
  • A clipboard-based KV store
  • A Markdown dashboard generator
  • A "CMD database engine" with indexing

sHEL feels like:

  • Redis
  • Jinja2
  • SQLite
  • A templating engine
  • A dashboard generator
  • And a macro system

...all fused into one.

Overview¶

What sHEL provides

The sHEL documentation system is organized into five main sections:

  • Symbols Archive — Character reference, Base64 alphabet, stereograms
  • XML Project — Literal-safe cmd.exe data handling, Base64, clipboard automation
  • Projects — Individual sHEL ecosystem projects: Countku, CAPTCHA, Regedited, Macrohard
  • Wikispace — Systems administration, networking, security, AI tooling
  • Deep Hole — Curated external resources and cultural references
